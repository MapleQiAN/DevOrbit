# DevOrbit · 开发者数字轨道仪表盘

> 把你散落在 GitHub / 刷题平台 / 本地笔记里的碎片努力，聚合成一份可视化的「数字履历」。

DevOrbit 是一个面向个人开发者 / 学生 / 技术从业者的开源仪表盘，它会自动聚合你在多个平台上的开发行为数据，构建时间序列 & 维度化的分析视图，帮助你回答几个常见却很难量化的问题：

- 「我这一年到底写了多少代码？」
- 「我最近到底在卷算法还是在卷 Web / AI？」
- 「我的成长节奏，是不是在某些时间段突然断档？」
- 「写在 GitHub / 刷题平台 / 本地笔记里的东西，能不能合起来看？」

DevOrbit 不是一个刷数据的「绩效面板」，而是一个面向**长期成长**的数字足迹聚合工具。

---

## ✨ 核心功能特点

### 1. 多数据源聚合 · 一处查看

目前规划支持三类数据源：

1. **GitHub 活动**
   - 每日 / 每周：
     - `commit` 数
     - `PR` 数
     - `issue` 活动
     - `star` 增长
   - 仓库维度：
     - 仓库列表 & 活跃度
     - 语言占比
     - 仓库创建/首个 star/首次 merged PR 等关键事件

2. **刷题平台记录（优先支持 LeetCode 导出）**
   - 按日期统计做题数量
   - 难度分布（Easy / Medium / Hard）
   - 题型 / 标签分布（双指针、DP、图论等）
   - 里程碑事件：
     - AC 数达到 100 / 300 / 500 题等

3. **本地学习笔记（Obsidian / Markdown 目录）**
   - 指定本地笔记根目录
   - 统计：
     - 每日/每周写作活跃度
     - 文件数 / 估算字数
     - 按 tag / 路径的主题分布（如：算法 / Web / 随笔）

> 设计目标：  
> 把「写代码」「刷题」「记笔记」视作同一条成长时间线上的不同维度。

---

### 2. 可视化仪表盘 · 一眼看出节奏

前端基于 Vue 3 + ECharts，提供多种视图：

- **日历热力图**
  - 把 GitHub 活动 + 刷题 + 笔记合并成统一的「活跃热度」日历
  - 支持按数据源/维度过滤
- **雷达图（技能维度画像）**
  - 预设若干技能维度，例如：
    - 算法 / Web / AI / 系统 / 工具
  - 通过仓库语言、topic、刷题标签、笔记 tag 做粗粒度归类
  - 得出一份「当前阶段的技能配置雷达图」
- **时间序列图**
  - 按日/周/月查看：
    - commit 数量变化
    - 刷题数量变化
    - 笔记活跃度变化
- **时间线视图（Timeline）**
  - 自动挖掘「关键时刻」，例如：
    - 「某仓库获得第一个 star」
    - 「第一次向开源项目提交 PR 并被合并」
    - 「刷题数达到新的里程碑」
    - 「开始写某一类主题的笔记」

---

### 3. 报告生成 · 月报 / 季报 / 年报

DevOrbit 提供一键生成「成长报告」功能，支持：

- **自定义时间范围**
  - 月报 / 季报 / 年报
  - 或指定开始/结束日期
- **报告内容结构（示例）**
  - 总览：
    - 时间段内总 commit 数 / 刷题数量 / 新增笔记数
    - 最活跃的一周 / 一天
  - GitHub 维度：
    - 最活跃仓库 Top 3
    - 新增 star、PR、issue 分布
    - 语言使用占比变化（如：这一季度 TypeScript 比例上升）
  - 算法 / 刷题维度：
    - 题目难度 & 题型分布
    - 「最近重点突破」领域
  - 学习笔记维度：
    - 新增笔记数量
    - 主要主题 / tag 统计
- **导出格式**
  - Markdown：方便贴到博客 / 知识库
  - PDF：适合用于汇报 / 简历附件 / 指导教师汇报等
- **可选：LLM 文本总结（可配置）**
  - 在本地或通过自定义接口调用 LLM
  - 输出更自然的「人话总结」段落

---

### 4. 成长对齐 & 自我复盘辅助

DevOrbit 的目标是成为你做自我复盘时的「数据底座」，而不是给你一个冷冰冰的数字墙。

典型使用场景：

- 写简历 / 年度总结时：
  - 用报告中的数据 & 时间线辅助回忆
  - 提取「关键项目」「能力提升点」的支撑证据
- 职业规划 / 竞赛答辩：
  - 提供可视化图表作为 PPT 的素材
  - 用「数据 + 时间」说明你的行动与成长不是嘴上说说
- 心态管理：
  - 识别「中长期空窗期」  
  - 能够看到「真正稳定坚持下来的东西」

## 🧱 技术栈与架构概览

### 后端

- **框架**：FastAPI
- **语言**：Python 3.10+
- **数据库**
  - 开发环境：SQLite
  - 生产环境：PostgreSQL
- **ORM & 迁移**
  - SQLAlchemy 2.x
  - Alembic
- **任务调度 / 异步任务**
  - 轻量模式：APScheduler
- **第三方集成**
  - GitHub OAuth 登录
  - GitHub REST API（仓库、events、统计等）
  - 刷题平台导出文件解析（优先 LeetCode）
- **报告生成**
  - Jinja2 模板渲染 Markdown
  - HTML / Markdown 转 PDF 的工具链（如 WeasyPrint 等）

### 前端

- **框架**：Vue 3 + TypeScript
- **构建工具**：Vite
- **状态管理**：Pinia
- **路由**：Vue Router
- **UI 组件库**：Naive UI / Element Plus（最终以项目为准）
- **图表**：Apache ECharts

### 部署

- **docker-compose 一键部署**
  - `api`：FastAPI 服务
  - `web`：前端静态资源（nginx）
  - `db`：PostgreSQL（生产推荐）
- 支持「开发模式」下的轻量运行：
  - FastAPI + SQLite + 前端 dev server 即可使用

## 🐳 Docker 一键部署

DevOrbit 现已支持完整的 Docker 一键部署方案！

### 快速开始（3 步）

```bash
# 1. 复制环境配置
cp .env.example .env

# 2. 编辑 .env，填入 GitHub OAuth 凭证
nano .env

# 3. 一键启动
docker-compose up -d
```

**完成！** 应用现在运行在：
- 🌐 前端: http://localhost
- 🔌 后端 API: http://localhost:8000
- 📚 API 文档: http://localhost:8000/docs

### 使用启动脚本

```bash
# 开发模式启动
./start.sh

# 生产模式启动
./start.sh --prod

# 重新构建并启动
./start.sh --build --logs
```

### 使用 Makefile

```bash
# 快速启动
make quick-start

# 启动服务
make up

# 查看日志
make logs
```
